The Core Concept: What is Polymorphism?
In the simplest terms, Polymorphism means "many forms" (from the Greek poly meaning many, and morph meaning form).

As explained in the lecture, think of it like a person in real life. A person named Rahul exists as a single entity, but he takes on many different forms depending on the context: to his parents, he is a son; to his wife, he is a husband; to his child, he is a father.

In C++, polymorphism allows functions or operators to behave differently based on the object they are acting upon or the arguments they are given.

It is broadly divided into two types:

1. Compile-Time (Static) Polymorphism: The compiler knows exactly which function to call before the program even runs. This is achieved via Function Overloading and Operator Overloading.

2. Run-Time (Dynamic) Polymorphism: The program decides which function to call while it is actively running, achieved via Virtual Functions (Function Overriding).

Compile- time polymorphism has two types:
1. function overloading
2. operator overloading

Run-time polymorphism is achieved using Function Overriding and the mighty virtual keyword.
